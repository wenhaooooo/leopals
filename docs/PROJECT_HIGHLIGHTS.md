# 🦁 LeoPals - 项目亮点文档

## 一、项目概述

LeoPals 是一个面向高校师生的垂直领域智能服务平台，整合校园非结构化数据与校务系统 API，通过 RAG（检索增强生成）和 **多智能体协作架构** 提供低延迟、防幻觉的问答与办事服务。

---

## 二、核心技术亮点

### 1. RAG 检索增强生成系统

#### 1.1 混合检索策略
- **向量检索**：基于 nomic-embed-text 模型生成 768 维嵌入向量，使用 PostgreSQL + pgvector 构建 HNSW 索引
- **BM25 全文检索**：传统关键词检索作为补充
- **RRF 融合算法**：Reciprocal Rank Fusion 融合两种检索结果，提升召回率

#### 1.2 Query Rewriting（查询重写）
- 使用 LLM 将口语化查询转换为精确检索关键词
- 支持模糊查询扩展，例如："奖学金怎么申请" → "奖学金申请条件流程"

#### 1.3 Contextual Compression（上下文压缩）
- 根据用户查询动态过滤文档内容
- 去除冗余信息，保留关键内容，减少 LLM 输入负担

#### 1.4 语义缓存层
- 基于 MD5 哈希的查询缓存机制
- 缓存有效期 5 分钟，最大缓存 100 条
- 显著减少重复查询响应时间

#### 1.5 文档版本控制
- 支持文档更新和历史回溯
- 自动记录版本号、内容哈希和更新时间

### 2. 多智能体协作架构

#### 2.1 智能体体系
```
用户请求
    │
    ▼
┌──────────────────┐      ┌──────────────────────┐
│  Orchestrator    │◄────►│   Message Bus       │
│   (调度智能体)   │      │   (消息总线)        │
└────────┬─────────┘      └──────────────────────┘
         │
    ┌────┴────┬────────────┬─────────────┬─────────────┐
    ▼         ▼            ▼             ▼             ▼
┌────────┐┌─────────┐ ┌──────────┐ ┌───────────┐ ┌──────────┐
│  QA    ││Schedule │ │Emotional │ │Knowledge  │ │Assistant │
│ Agent  ││  Agent  │ │  Agent   │ │  Agent    │ │  Agent   │
│ (问答) ││(日程)   │ │(情感支持)│ │(知识库)   │ │(助手)    │
└────────┘└─────────┘ └──────────┘ └───────────┘ └──────────┘
```

#### 2.2 智能体职责划分

| 智能体 | 职责描述 | 核心能力 |
|--------|---------|---------|
| **Orchestrator** | 意图识别、任务分发、结果汇总 | 意图分类、智能体选择、多智能体协作 |
| **QAAgent** | 事实性问答 | RAG检索、精准回答 |
| **ScheduleAgent** | 课表查询、日程管理 | 课表API、日历集成、提醒设置 |
| **EmotionalAgent** | 情感陪伴、心理支持 | 情感分析、共情回应 |
| **KnowledgeAgent** | 深度文档理解 | 政策解读、多模态分析 |
| **AssistantAgent** | 复杂任务规划 | 任务分解、步骤执行、智能体协调 |
| **FunctionCallingAgent** | 业务系统闭环 | 动态技能系统、工具调用、参数提取自动化 |

#### 2.3 动态技能注册系统

```
Agent (智能体)
    │
    ├─→ Skill Registry (技能注册表)
    │   ├─→ ScheduleSkill (课表技能)
    │   ├─→ GradeSkill (成绩技能)
    │   ├─→ ClassroomSkill (教室技能)
    │   └─→ NotificationSkill (通知技能)
    │
    └─→ Skill Loader (动态加载)
        ├─→ 本地技能 (.py 文件)
        ├─→ 远程技能 (Git 仓库)
        └─→ MCP Server
```

**核心特性**：
- **插件化架构**：技能以插件形式管理，支持热加载
- **类型安全**：基于 Pydantic 的参数校验
- **异步执行**：支持高并发技能调用
- **多源加载**：支持本地文件、Git 仓库、MCP Server
- **生命周期管理**：支持技能的启用、禁用、注销
- **上下文传递**：支持用户 ID、会话 ID 等上下文信息
- **LangChain 集成**：技能自动转换为 LangChain 工具，无缝集成到 Function Calling

**已实现技能**：
| 技能名称 | 功能 | 类别 |
|---------|------|------|
| `schedule_query` | 查询课表 | schedule |
| `grade_query` | 查询成绩 | academic |
| `classroom_search` | 搜索空教室 | facility |
| `notification_set` | 设置提醒 | notification |

**技术优势**：
- **扩展性强**：新增技能无需修改核心代码，只需继承 BaseSkill
- **维护性好**：技能独立开发、测试、部署
- **灵活性高**：支持运行时动态加载和热重载
- **集成度高**：与 LangChain 工具体系无缝对接

#### 2.4 协作模式

- **顺序协作**：智能体依次执行，前一个智能体的输出作为后一个的输入
- **并行协作**：多个智能体同时执行，结果汇总后统一返回
- **嵌套协作**：AssistantAgent 分解任务，协调多个智能体协作完成复杂任务

#### 2.5 消息总线机制

- **解耦通信**：智能体之间通过消息总线通信，降低耦合度
- **同步/异步支持**：支持同步调用和异步广播
- **消息格式规范**：统一的请求/响应格式，便于扩展

### 3. LangGraph Agent 状态机

```
用户提问 → Router节点 → 工具调用/知识检索 → 生成回答 → 结束
```

- **多步决策**：基于状态机的智能路由，根据问题类型选择处理方式
- **工具调用**：支持课表查询、提醒管理等工具的自动调用
- **流式输出**：SSE 实时流式响应，区分思考过程与最终回答

---

### 4. 多模态支持

- **图片识别**：课程表图片 → 结构化数据（使用 OCR）
- **语音交互**：STT（语音转文字）+ TTS（文字转语音）

---

### 5. 智能日程管理

- **课程表同步**：支持从教务系统自动拉取或图片识别导入
- **智能提醒**：作业截止、考试安排自动提醒
- **冲突检测**：课表与活动冲突自动检测

## 五、性能优化策略

### 5.1 检索优化
- **向量索引**：HNSW 索引加速向量检索
- **语义缓存**：减少重复查询
- **查询重写**：提升检索精度

### 5.2 响应优化
- **流式输出**：SSE 实时响应，降低感知延迟
- **异步处理**：FastAPI 异步端点，提高并发能力

### 5.3 数据库优化
- **索引优化**：合理的索引设计
- **批量操作**：减少数据库交互次数

### 5.4 智能体优化
- **按需调用**：根据意图选择智能体，避免不必要的计算
- **并行执行**：多个智能体可并行工作，提升处理效率

## 十、多智能体架构优势

| 维度 | 单一智能体 | 多智能体 |
|------|-----------|---------|
| **专业化程度** | 通用能力，无专业深度 | 各智能体专注特定领域 |
| **代码可维护性** | 单一大文件，难以维护 | 模块化，易于维护 |
| **扩展性** | 新增功能需要修改整体逻辑 | 新增智能体即可扩展 |
| **性能** | 单点处理，无法并行 | 可并行执行多个智能体 |
| **容错性** | 单点故障导致整体不可用 | 部分智能体故障不影响其他功能 |
| **资源利用** | 资源占用固定 | 按需调用智能体，资源利用率高 |
